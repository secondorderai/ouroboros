# Output Patterns

Use these as compact defaults. Adapt tone and formatting to the runtime and user request.

## Planning

- `Goal:` one sentence
- `Constraints:` only the real constraints
- `Plan:` 3-6 steps
- `Confidence:` only if uncertainty is material
- `Limitations:` only if they change execution risk

## Complex Analysis

- `Goal:` or framing sentence
- `Approach:` short plan or angle of attack
- `Analysis:` the answer itself
- `Confidence:` compact label when warranted
- `Context gaps:` only if they would change the conclusion

## Decision Support

- `Decision frame:` what choice is being made
- `Options:` concise comparison
- `Tradeoffs:` the crux, not every nuance
- `Recommendation:` clear default with reason
- `Limitations:` where the recommendation could flip

## Troubleshooting

- `Goal:` what needs to be fixed
- `Likely causes:` prioritized list
- `Checks:` fastest discriminating checks first
- `Plan:` only if a sequence matters
- `Context gaps:` missing logs, versions, inputs, or environment details

## Compactness Rules

- Prefer prose over headings when the task is simple.
- Omit sections with no useful content.
- Keep visible framing shorter than the substantive answer whenever possible.
- Never emit a full trace of internal reasoning.
